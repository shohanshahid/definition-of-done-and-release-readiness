# Quality Gates

## Definition of done checklist
- Acceptance criteria met and validated  
- Tests executed and results recorded  
- No known critical defects  
- Logging and monitoring considered  
- Documentation updated where needed  
- Code review completed  
- Deployment notes captured  

## Release readiness checklist
- Rollback plan confirmed  
- Environment configuration verified  
- Data migration steps validated if relevant  
- Observability checks confirmed  
- Stakeholder sign off recorded  
- Post release verification steps defined  

## Notes
Use these gates as release conditions, not suggestions. If a gate is not met, record the exception, owner, and next action.
